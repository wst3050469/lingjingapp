### 通义千问（数据与安全官）

1. 【30天行动计划】  
① **部署记忆审计流水线**：第3天上线日志采集器（asyncpg监听`memory_events`表INSERT/UPDATE），第7天完成PG审计视图（含操作人、时间、向量ID、原始文本哈希、共识校验码）；第15天接入通义千问偏见审计API（每条记忆调用≤2次/天，限前1000条v1-v7记录）。  
② **实施向量存储安全加固**：第5天将pgvector索引迁移至独立schema `secure_vector`；第10天启用Row-Level Security策略，仅允许`memory_reader`角色SELECT（需JWT鉴权+IP白名单）；第20天完成嵌入向量脱敏测试（nomic-embed-text输入前截断首尾15字符，误差率<0.8%）。  
③ **建立记忆生命周期看板**：第12天部署Grafana（端口3001），监控7项指标：记忆写入QPS、向量维度合规率（必须=768）、共识校验失败数/日、未触发审计的写入占比（目标≤0.3%）、单条记忆最大长度（硬限128KB）、pgvector索引碎片率（>25%自动VACUUM）、Ollama embedding超时率（阈值3%）。  

2. 【核心建议】  
**所有记忆写入必须强制双签：① 通义千问生成语义指纹（SHA3-384）+ ② DeepSeek输出逻辑一致性校验码（CRC64），缺一不可入库。否则拒绝写入。**

# 灵境项目第一阶段（30天）执行计划

## 1. 【产品定位】
灵境是一个基于多模型AI共识的**数字记忆存证与智能协作平台**，为**AI开发者、内容创作者及有数字资产存证需求的个人/企业**提供可追溯、防篡改、高可用的AI交互记忆存储与检索服务，解决数字内容创作过程**不可信、易丢失、难追溯**的核心痛点。

## 2. 【MVP功能清单】
- **P0 (必须上线)**：
    1. **记忆存储与检索核心API**：支持文本上传、向量化存储、基于语义的相似性检索。
    2. **记忆三层结构**：原始层（原文）、索引层（向量+标签）、共识层（操作日志与哈希）。
    3. **基础审计日志**：记录所有记忆的写入、读取操作（操作者、时间、内容哈希）。
    4. **多模型调用框架**：可异步调用9个AI模型API，并记录调用详情。
- **P1 (30天内完成)**：
    1. **记忆标签化与分类**：对v1-v7历史记录完成自动标签化，支持按标签过滤检索。
    2. **伦理风险扫描**：自动扫描新记忆中的“温情主义”等风险词并告警。
    3. **成本监控看板**：按模型统计API调用次数与费用。
- **P2 (30天后规划)**：
    1. **知识图谱可视化查询**。
    2. **用户社区与内容共创功能**。
    3. **商业化付费功能（如高级存证、批量处理）**。

## 3. 【技术架构方案】
在现有 `FastAPI + PostgreSQL 16 + pgvector` 基础上，进行以下补充与整合：
- **统一数据层**：
    - 主库 `lingjing_primary`：运行所有业务表。
    - 独立Schema `secure_vector`：存放pgvector索引与向量数据，启用Row-Level Security。
    - 核心表：`raw_memory`（原始层）、`memory_index`（索引层）、`audit_log`（统一审计日志，合并操作、伦理、校验日志）。
- **服务层**：
    - 统一API网关（由FastAPI实现），集成记忆存储、检索、模型调度、审计流水线。
    - **取消“中央协调器”重复建设**，模型调度作为网关内置模块。
- **部署与运维**：
    - 采用Docker Compose在两台服务器上部署标准化环境（Server A主服务，Server B冷备）。
    - 使用Nginx做反向代理和负载均衡（预留）。
    - 部署Grafana（端口3001）监控关键指标。

## 4. 【30天里程碑】

    - D1-D3：完成两台新服务器的标准化环境部署（Docker, PostgreSQL, pgvector, FastAPI）。
