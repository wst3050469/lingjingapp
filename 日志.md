### v1.64.6 — 文件变更自动处理功能优化（UI分离保存按钮 + 实时生效） ✅

**2026-05-31**
- **任务**: 优化 QuestTab 中"文件变更自动处理"设置的UI和生效机制
- **根因**: 现有代码在点击单选按钮时即调用 `saveKey()` 保存到磁盘，无独立"保存"按钮；且设置变更后 QuestConversation 只在挂载时加载一次配置，无法实时生效
- **修改文件 (2个)**:
  1. `packages/renderer/src/components/settings/tabs/QuestTab.tsx`:
     - ✅ 从单选 `onClick` 中移除 `saveKey()` 调用，改为仅更新本地 `fileChangeBehavior` + 标记 `fcbDirty = true`
     - ✅ 新增 `handleSaveFileChangeBehavior` 函数：调用 `saveKey()` 持久化 + `dispatchEvent(CustomEvent)` 通知 QuestConversation
     - ✅ 在单选按钮组下方添加"保存配置"按钮（仅 `fcbDirty` 为 true 时显示）
  2. `packages/renderer/src/components/quest/QuestConversation.tsx`:
     - ✅ 新增 `useEffect` 监听 `quest:file-behavior-changed` 自定义事件
     - ✅ 收到事件后立即更新 `fileChangeBehavior` state，触发自动处理逻辑（无需重新挂载组件）
- **验证**: ✅ renderer 侧 TypeScript 编译零错误（`npx tsc --noEmit -p packages/renderer/tsconfig.json`）
- **潜在风险**: ✅ **极低** — CustomEvent 只在保存时发送一次，不影响现有 useEffect 链；保存按钮仅在状态脏时显示，不影响现有 UI
- **测试点**:
  1. 打开设置 → Quest 模式 → 文件变更处理 → 选择选项 → 不点保存 → 关闭设置 → 再次打开 → 显示原设置
  2. 选择后点击"保存配置" → 底部显示"已保存" → 关闭设置 → 再次打开 → 显示已保存的选项
  3. 设置为 "auto-accept" → 保存 → Quest 模式下 AI 创建文件 → 自动接受
  4. 设置从 "auto-accept" 改为 "ask" → 保存 → AI 创建文件 → 保持 pending 状态
  5. 多次切换选择 → 每次都点保存 → 每次都即时生效
