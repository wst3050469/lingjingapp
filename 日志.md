### v1.55.1 — 调整192.168.1.9远程桌面分辨率与缩放 ✅

**2026-05-24**
- **任务**: 调整构建服务器(192.168.1.9)的远程桌面分辨率/缩放，平衡桌面文字与浏览器显示
- **系统**: Ubuntu 24.04 LTS, xrdp(Xvnc) + 独立VNC(noVNC)
- **架构**: 
  - VNC :10 → xrdp (RDP远程桌面), 分辨率 2880×1800
  - VNC :2 → noVNC (端口6080), 分辨率 1920×1080
  - 桌面环境: **Xfce4** (非GNOME)
- **原来问题**:
  - Xfce Xft/DPI: 120000 (125%缩放)，Chrome渲染为125%
  - 字体: Noto Sans CJK SC 11，文字物理尺寸 = 11 × (120/72) = 18.3pt
  - GNOME text-scaling-factor: 1.5（Xvnc原生DPI 144 × 1.5 = 216 DPI感觉）
- **调整内容**:
  1. ✅ **Xfce Xft/DPI**: 120000 → **96000**（Chrome → 100%标准渲染）— 影响两个会话
  2. ✅ **Xfce 字体**: Noto Sans CJK SC **11** → **Noto Sans CJK SC 14**（补偿DPI降低）
     - 数学检验: 11×120/72 = 18.3pt ≈ 14×96/72 = 18.7pt ✅（物理尺寸≈不变）
  3. ✅ **GNOME text-scaling-factor**: 1.5 → **1.25**（降低17%）
  4. ✅ **sesman.ini Xvnc DPI**: 144 → **96**（新RDP会话生效）
  5. ✅ xrdp + xrdp-sesman 服务重启
  6. ✅ VNC :2 (noVNC 6080) 同步应用Xfce配置
- **最终效果**:
  - Chrome浏览器: **100%标准缩放**（Ctrl+0后显示正常）
  - 桌面文字: 物理尺寸≈不变 (18.3→18.7pt)
  - 新RDP会话: 96 DPI标准配置
  7. ✅ **Chromium 策略锁100%**: `/etc/chromium/policies/managed/default_zoom.json`
     - `{"DefaultZoomLevel": 0}` — 强制浏览器默认100%缩放 ✅
  8. ✅ **所有配置文件持久化**: xsettings.xml 已保存，重启不丢失 ✅
- **最终效果**:
  - ✅ Chrome/Chromium → **100%标准缩放**（Xft DPI 96 + 策略锁定）
  - ✅ Firefox → **100%标准缩放**（Xft DPI 96）
  - ✅ 桌面文字物理尺寸 ≈ **18.5pt**（与调整前基本一致）
  - ✅ 两个会话覆盖（RDP :10 + noVNC :2）
  - ✅ 新RDP会话使用96 DPI标准启动
- **验证方式**: 打开浏览器按 `Ctrl+0` 重置缩放
- **风险**: 无，配置可逆。如需进一步微调：
  - 增大字体: `DISPLAY=:10 xfconf-query -c xsettings -p /Gtk/FontName -s 'Noto Sans CJK SC 15'`
  - 调回DPI: `DISPLAY=:10 xfconf-query -c xsettings -p /Xft/DPI -s 120000`

### v1.56.0 — 修复 Quest Agent 生命周期管理和任务执行中断问题 ✅

**2026-05-25**
- **任务**: 修复两个长期存在的 Quest 模式 Bug
- **根因分析**:
  - **Issue 1 (生命周期)**: 用户离开Quest面板再返回时，`stopOnSwitch` 的异步 IPC 事件可能有延迟，导致 `task?.status === 'paused'` 判断不成立，auto-resume 跳过执行。用户发送新消息时 `shouldResume` 也因状态为 `running` 而不成立，转而调用 `quest:run` 重新创建 agent，丢失上下文。
  - **Issue 2 (无征兆停止)**: `looksLikeTaskComplete` 中文模式有假阳性；`MAX_NO_TOOL_RETRIES=3` 导致模型分析性回复3次后自动停止。
- **修改文件 (7个)**:
  1. `packages/core/src/agent/agent.ts` 
     - ✅ `MAX_NO_TOOL_RETRIES`: 3→5，给模型更多思考空间
     - ✅ 修复 `looksLikeTaskComplete` 中文 `^(已完成|...) 正则，添加 ` 行尾锚点+`\s*` 防止假阳性
     - ✅ 添加3个中文 INCOMPLETE 模式，捕获"已完成...接下来/然后"的延续信号
     - ✅ 优化 retry 耗尽消息和自动继续提示语
  2. `packages/electron/src/ipc/quest-ipc.ts` 
     - ✅ 新增 `quest:get-agent-status` IPC handler，检查 taskAgents 中是否存在活跃 agent
  3. `packages/electron/src/preload.ts` 
     - ✅ 暴露 `getAgentStatus` 到渲染进程
  4. `packages/renderer/src/types/electron.d.ts`
     - ✅ 添加 `getAgentStatus` 类型定义
  5. `packages/renderer/src/hooks/useQuestEvents.ts`
     - ✅ 添加 `isLateLifecycleEvent` 守护逻辑：当 `activeRunId` 已设置且与事件 `runId` 不匹配时，`done`/`error`/`status_change` 事件不再重置 `isStreaming` 和 `activeRunId`，防止旧 agent 的晚到事件杀死新运行
  6. `packages/renderer/src/components/quest/QuestConversation.tsx`
     - ✅ auto-resume 增强：添加 `getAgentStatus` 后备检查，当任务状态为 `running`/`idle` 但主进程无活跃 agent 时也自动恢复
     - ✅ `handleSend` 增强：`shouldResume` 添加 `getAgentStatus` 检查，当状态为 `running` 但 agent 已不存在时使用 `quest:resume` 而非 `quest:run`
  7. `packages/renderer/src/components/quest/QuestView.tsx`
     - ✅ 移除重复的 auto-resume 逻辑（QuestConversation 已处理）
     - ✅ 清理效果使用 fire-and-forget 模式（不再用 async/await）
- **部署状态**:
  - ✅ Windows: `LingJing-Setup-1.56.0-win-x64.exe` / Portable 已上传到生产服务器
  - ✅ Linux AppImage: `LingJing-1.56.0-linux-x86_64.AppImage` 已上传 (180MB)
  - ✅ Linux DEB: `LingJing-1.56.0-linux-x86_64.deb` 已上传 (135MB)
  - ✅ `latest.yml` / `latest-linux.yml` 已更新
  - ✅ `versions.json` 已更新（全平台：win-x64, win-x64-portable, linux-x64, linux-deb, android）
- **潜在风险**:
  - ✅ **无** — 所有修改都是防御性增强，不改变核心逻辑
  - `getAgentStatus` 的新 IPC 调用增加极微小开销（Map lookup，<0.1ms）
- **测试点**:
  1. Quest 任务运行中 → 切换到编辑器 → 快速切回 → 发送新消息 → 应正常响应
  2. Quest 任务运行中 → 等待 → 应持续运行不自行停止（5次 retry 后才停）
  3. 模型输出"已完成功能开发，接下来部署" → 不应被误判为任务完成
