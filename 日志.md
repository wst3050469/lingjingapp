## [2026-06-13] v1.73.48: executor.js MODULE_NOT_FOUND 崩溃终极修复

### 问题
v1.73.47 的 agent.ts 动态 import 修复被 `index.ts` 的静态重导出短路，用户安装 NSIS 包后仍崩溃：
```
Error [ERR_MODULE_NOT_FOUND]: Cannot find module '...executor.js' imported from '...agent.js'
```

### 双重根因 → 三重根因（深度排查发现）
| # | 根因 | 位置 | 严重度 |
|---|------|------|:--:|
| A | `index.ts` L27 `export { ToolExecutor } from './tools/executor.js'` 静态重导出 | `desktop/core/src/index.ts` | 🔴 P0 |
| A2 | `index.ts` L11 `export { AgentFactory }` → `agent-factory.js` L2 `import { ToolExecutor }` 同样静态导入 | `index.ts` + `agent-factory.js` | 🔴 P0 |
| B | v1.73.47 NSIS 安装包未包含 ASAR 热修补 | 生产部署 | 🔴 P0 |

#### 根因 A2 详解（第二轮排查发现）
`agent-factory.js` 第2行有 `import { ToolExecutor } from '../tools/executor.js'`，同样在模块链接阶段崩溃。该模块通过 `index.ts` 的 `export { AgentFactory }` 被强制加载。`AgentFactory` 全项目零消费者。

#### 根因 A 详解
agent.ts 已改为动态 `import()` + FallbackExecutor（正确），但 `index.ts` 的静态重导出 `export { ToolExecutor } from './tools/executor.js'` 在 ESM **模块链接阶段**即强制解析 executor.js。调用链：
```
Electron → main.js → fusion-ipc.ts/openspace-ipc.ts 
  → import from '@codepilot/core'
    → index.js 静态模块链接
      → export { ToolExecutor } from './tools/executor.js' ← 💥
```
agent.ts 的 try-catch 从未执行，崩溃发生在模块链接阶段。

## [2026-06-12] 版本检测数据源同步修复 (已完成)

### 问题分析
桌面端和移动APP已安装最新版本(v1.73.34)，但版本检测仍报错/提示更新。

### 根因
生产环境存在多个 versions.json / latest.yml 文件分布在 5 个不同位置，各自维护且版本号不一致。

cloud-server 的 `readVersionInfo()` 优先读取 `/var/www/lingjing/versions.json`（修复前=1.72.33），导致 `/api/latest` 返回过期版本号。所有客户端（桌面 httpCheckVersion、移动端 UpdateChecker）获取到错误数据。

### 版本文件修复

| 文件 | 修复前 | 修复后 |
|------|--------|--------|
| `/var/www/lingjing/versions.json` | 1.72.33 | **1.73.34** |
| `/var/www/lingjing/latest.yml` | 1.72.29 | **1.73.34** |
| `/var/www/html/versions.json` | 1.73.34 | 1.73.34 (不变) |
| `/var/www/downloads/latest.yml` | 1.73.34 | 1.73.34 (不变) |
| `/opt/lingjing/update-server/data/versions.json` | 1.72.20 | **1.73.34** |
| repo `services/update/data/versions.json` | 1.54.0 | **1.73.34** |

### 代码修复
- **`services/update/app.js`** (update-server, 端口3000): 修复 `/api/latest` 始终返回 `hasUpdate: true` 的缺陷。新增 `current` 查询参数支持客户端版本对比。增强 `latest.yml` 生成兼容性。

- **`services/backend/server.js`** (cloud-server): 修复 `readVersionInfo()` 搜索路径（新增 `/var/www/lingjing/versions.json` 和 `/var/www/html/versions.json` 为首选路径），添加 `compareVersions()` 函数，`/api/latest` 支持 `?current=` 参数。
