### v1.64.11 — maxTurns 配置三方不一致导致用户设置不生效的修复 ✅ 🔧

**2026-06-02**
- **任务**: 排查"Reached maximum number of turns"报错，定位到 maxTurns 配置在 Schema/UI/默认值三方不一致
- **根因**: 
  - Schema (`schema.js`) 的 `min(500)` 导致用户保存的 `maxTurns: 50` 被 Zod 静默拒绝，回退到默认值 500
  - UI 默认值硬编码为 50，与实际默认值 500 不一致（用户不保存也看到 50）
- **修改**:
  - `packages/core/dist/config/schema.js`: `min(500)` → `min(1)`，允许用户灵活设置（Quest 模式有独立 `Math.max(500)` 保护）
  - `packages/renderer/src/components/settings/tabs/SessionTab.tsx`:
    - L82: 默认值 `?? 50` → `?? 500`（对齐 schema 默认值）
    - L114: 校验范围 `10~500` → `1~2000`（对齐 schema 约束）
    - L191-192: input `min=10 max=500` → `min=1 max=2000`
    - L187: 描述文字更正
- **安全防护**:
  - Quest 模式 (`quest-ipc.ts`): `Math.max(config.maxTurns ?? 500, 500)` — 始终至少 500 轮
  - Agent 模式 (`agent-ipc.ts`): 使用 `config!.maxTurns` — 遵循用户配置
