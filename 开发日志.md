## 2026-05-30 APK下载链路全面修复

### 问题诊断
1. CDN WAF拦截: 阿里云CDN/WAF (120.55.5.220) 仅允许特定路径，/downloads/*.apk被拦截返回404
2. OSS APK限制: 阿里云OSS禁止直接分发.apk文件（ApkDownloadForbidden错误）
3. 云服务器302重定向: cloud-server对公网请求返回302→/api/v1/app/apk/，该路径被CDN WAF拦截(403)
4. APK文件路径错误: _APK_DIR指向/home/liuhui/灵境/app/但文件存储在/home/liuhui/灵境/server/app/

### 修复内容

#### 1. update-server (app.js)
- 新增/downloads/:filename静态文件下载路由
- 新增/downloads/get-mobile-apk路由（无扩展名，绕过CDN文件拦截）
- 将特定路由放在通用路由之前，避免被catch-all捕获

#### 2. cloud-server (app_version.py)
- 移除公网请求302重定向逻辑
- 所有请求直接流式返回FileResponse
- APK文件从/var/www/html/复制到正确的_APK_DIR(/home/liuhui/灵境/app/)

#### 3. 版本文件
- 修复git stash冲突导致的versions.json JSON损坏
- 更新所有mobile URL为 https://ide.zhejiangjinmo.com/api/v1/app/download/v1.64.1

### 验证结果
- Check-update API (CDN) → 200
- APK下载(CDN, 跟随重定向) → 200, 38MB有效APK
- 数据库v1.64.1已发布

### 关键发现
- 阿里云OSS禁.apk分发→通过API代理流式返回
- _APK_DIR = PROJECT_ROOT/app/ (不是server/app/)
- CDN路径: ide.zhejiangjinmo.com(主站) / lingjing.zhejiangjinmo.com(CDN分发)
