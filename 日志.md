## [2026-06-12] v1.73.36 sha512修复：版本检测"安装包校验失败" (已完成)

### 问题
1. 用户从 v1.72.31 升级检测到 v1.73.36，下载后报"安装包校验失败，文件可能已损坏"
2. 下载链接 `https://ide.zhejiangjinmo.com/downloads/灵境 Setup 1.73.36.exe` 中文编码问题

### 根因
`latest.yml` 和 `versions.json` 中 NSIS 安装包 (`灵境 Setup 1.73.36.exe`, 138MB) 的 sha512 值错误。
- **存储值**: `8b0b1167...` ← 实际是 **Portable 文件**的 hash
- **正确值**: `d5c14359...` ← NSIS 安装包的真实 hash

构建时误将 Portable 的 sha512 复制给了 NSIS 安装包条目，导致 electron-updater 下载后校验失败。

### 修复
| 操作 | 详情 |
|------|------|
| `latest.yml` | 修正 NSIS sha512 → `d5c14359...`，新增 Portable 条目，URL 中文编码 |
| `versions.json` ×3 | `/var/www/html/`, `/var/www/lingjing/`, `/opt/lingjing/update-server/data/` 修正 sha512 + URL 编码 |
| `latest.yml` 同步 | 同步到 `/var/www/lingjing/` 和 `/var/www/html/` |
| PM2 重启 | cloud-server, update-server, lingjing-update-server 全部重启 |

### 验证 (全部通过 ✅)
| 端点 | 结果 |
|------|------|
| cloud-server:8000 `/api/latest?current=1.72.31` | `hasUpdate:true, version:1.73.36` ✅ |
| cloud-server:8000 `/api/latest?current=1.73.36` | `hasUpdate:false` ✅ |
| update-server:3000 `/api/latest?current=1.72.31` | `hasUpdate:true, version:1.73.36` ✅ |
| lingjing-update:3002 `/api/latest?current=1.72.31` | `hasUpdate:true, version:1.73.36` ✅ |
| `latest.yml` HTTP | 200, 644 bytes ✅ |
| 下载链接 (URL编码) | 200, 143760437 bytes ✅ |
| NSIS sha512 匹配 | `d5c14359...` ✅ |
| Portable sha512 匹配 | `8b0b1167...` ✅ |
| versions.json 9个路径全同步 | 3 OK + 6 已同步 ✅ |

### 教训
构建安装包后，**必须用 `sha512sum` 重新计算每个文件的 hash**，不能从 electron-builder 输出中手工复制。建议在构建脚本中自动计算并写入 versions.json。

## [2026-06-12] 版本检测数据源同步修复 (已完成)

### 问题分析
桌面端和移动APP已安装最新版本(v1.73.34)，但版本检测仍报错/提示更新。

### 根因
生产环境存在多个 versions.json / latest.yml 文件分布在 5 个不同位置，各自维护且版本号不一致。

cloud-server 的 `readVersionInfo()` 优先读取 `/var/www/lingjing/versions.json`（修复前=1.72.33），导致 `/api/latest` 返回过期版本号。所有客户端（桌面 httpCheckVersion、移动端 UpdateChecker）获取到错误数据。

| `/var/www/html/versions.json` | 1.73.34 | 1.73.34 (不变) |
| `/var/www/downloads/latest.yml` | 1.73.34 | 1.73.34 (不变) |
