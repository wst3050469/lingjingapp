### Phase 96: Quest Agent 生命周期深度修复

**2026-05-23**
- **背景**: Quest Agent 在切换任务/离开Quest模式时，存在4个断裂点导致状态残留或无法恢复
- **断裂点1**: runId 过滤器丢弃了 `done` 和 `status_change` 生命周期事件
  - **修复**: `useQuestEvents.ts` — 添加 `isLifecycleEvent` 豁免，确保 done/status_change 不被 runId 过滤丢弃
  - 同时增强: error 事件增加 `setActiveRunId(null)` + `removeRunningTask()`；done 事件增加 `setActiveRunId(null)`
- **断裂点2**: 组件挂载时无自动恢复逻辑
  - **修复**: `QuestView.tsx` + `QuestConversation.tsx` — mount useEffect 检测 paused 状态任务自动调用 `quest.resume()`
- **断裂点3**: `handleSend` 始终调用 `quest:run` 而非 `quest:resume`
  - **修复**: `QuestConversation.tsx` — 智能判断: 有历史消息且状态为 paused/idle/failed 时调用 `resume()`，否则调用 `run()`
- **断裂点4**: `stopOnSwitch` 设置 `idle` 状态 + Resume 按钮条件过严
  - **修复**: `quest-ipc.ts` — `stopOnSwitch` 设置 `status='paused'`（原为 'idle'）
  - `QuestConversation.tsx` — Resume 按钮条件放宽为 `status==='paused' || status==='idle'`
- **涉及文件(4)**:
  - `packages/electron/src/ipc/quest-ipc.ts` (stopOnSwitch: idle→paused)
  - `packages/renderer/src/hooks/useQuestEvents.ts` (lifecycle豁免 + cleanup)
  - `packages/renderer/src/components/quest/QuestView.tsx` (mount auto-resume)
  - `packages/renderer/src/components/quest/QuestConversation.tsx` (mount auto-resume + smart dispatch + Resume放宽)
- **风险**: 低 — 所有修改为防御性增强，不影响正常 run 流程
- **验证**: 代码对比确认4个断裂点均已修复 ✅
