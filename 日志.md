## v1.73.171 - 键盘中文输入支持 (2026-06-23 14:30)

### 背景
v1.73.170 已知缺陷②：robotjs 的 `typeString`/`typeStringDelayed` 仅支持 ASCII 字符，中文等多字节字符无法输入。

### 修复方案
| 场景 | 方案 |
|------|------|
| ASCII 文本（如 "hello"） | 走 robotjs 原生 `typeString`，零开销 |
| 非 ASCII 文本（含中文/emoji 等） | Electron `clipboard.writeText()` + 模拟 Ctrl+V 粘贴 |

### 剪贴板粘贴流程
1. 保存当前剪贴板内容（文本+图片）
2. 写入目标文本到剪贴板
3. 模拟 Ctrl+V / Cmd+V
4. 等待粘贴完成（min 150ms，按文本长度动态计算）
5. `finally` 恢复原始剪贴板内容

### APP页面崩溃修复

#### 问题根因
`App.tsx` 中 `NavigationContainer` 的 `theme` prop 使用完全手动构造（含手动 `fonts` 定义），未基于 `DefaultTheme`/`DarkTheme` 展开。React Navigation v7 内部 `@react-navigation/elements` 的 `Text.js` 通过 `useTheme()` 获取主题后访问 `fonts.regular`，手动构造导致 `fonts` 为 `undefined`，触发 `Cannot read property 'regular' of undefined`。

**注意**: 此问题与 v1.73.156 根因完全一致，v1.73.156 的修复未在当前代码中生效（回归或被覆盖）。

#### 修复
```tsx
// Before: 完全手动构造
<NavigationContainer theme={{ dark: isDark, fonts: {...}, colors: {...} }}>

// After: 基于标准主题展开
<NavigationContainer theme={isDark
  ? { ...DarkTheme, colors: { ...DarkTheme.colors, ...customDarkColors } }
  : { ...DefaultTheme, colors: { ...DefaultTheme.colors, ...customLightColors } }
}>
```

## v1.73.161 - robotjs原生模块防御性加载 + asar.smartUnpack修复 (2026-06-22 17:30)

### 问题根因
robotjs 是 C++ 原生扩展，其 `.node` 二进制文件无法从 ASAR 压缩包加载。
同时，`desktop-control-ipc.ts` 中 `import robot from 'robotjs'` 是静态导入，
若原生模块缺失/版本不匹配，会导致整个 Electron 应用启动崩溃（MODULE_NOT_FOUND）。

### 修复方案（三层防护）

**第一层：asar.smartUnpack 自动解包**
- electron-builder v25 中 `asar.unpack` 配置已废弃，改用 `asar.smartUnpack: true`
- `smartUnpack: true` 自动检测并将 `.node` 原生模块从 ASAR 中解包到 app.asar.unpacked 目录

**第二层：延迟加载（Lazy Load）**

## v1.73.156 - App登录后页面崩溃修复 (2026-06-22 15:30)

### 问题
用户报告APP登录后出现"页面出错了 - Cannot read property 'regular' of undefined"错误

### 根因
`App.tsx` 中 `NavigationContainer` 的 `theme` prop 使用完全手动构造的方式，未基于 `DefaultTheme`/`DarkTheme` 展开。
React Navigation v7（@react-navigation/native v7.3.3 + @react-navigation/core v7.21.1）中，
`@react-navigation/elements` 的 `Text.js` 组件通过 `useTheme()` 获取主题后访问 `fonts.regular`，
当主题对象未基于标准主题构建时，内部合并逻辑可能导致 `fonts` 为 `undefined`。

### 修复
将自定义 theme 改为基于 `DefaultTheme`/`DarkTheme` 展开，仅覆盖 `colors` 字段：

```tsx
// Before: 完全手动构造 theme（含 fonts）
<NavigationContainer theme={{ dark: isDark, fonts: {...}, colors: {...} }}>

// After: 基于标准主题展开
<NavigationContainer theme={isDark 
  ? { ...DarkTheme, colors: { ...DarkTheme.colors, ...customColors } }
  : { ...DefaultTheme, colors: { ...DefaultTheme.colors, ...customColors } }
}>
```
